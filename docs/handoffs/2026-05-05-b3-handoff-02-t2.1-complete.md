## 技术风险待观察

- **T2.3 47 处 SQL 改写**是单点最大风险：SQL 字符串分散、每处语义不同、`RETURNING` 与事务语义迥异。建议每改完一组（按业务块 5-10 处）立即跑测试；测试若因 mock 改造失败，要回改测试适配而非绕过。
- **ReadOnly tx**：SQLite 没有独立 `FOR UPDATE`，事务本身是串行化的。老代码 `select ... for update` 的语义在 SQLite 里自然达成（`BEGIN IMMEDIATE` / `BEGIN EXCLUSIVE`）；但并发读时要改写。
- **big.Int 序列化**：`numeric(20,0)` → TEXT 后，业务代码 Scan/Value 要走 `big.Int.SetString` / `.String()` 不是 `int64`。现有代码用的是 `big.Int` 吗？T2.3 要 spot check。
